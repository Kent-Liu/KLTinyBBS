# KLTinyBBS 使用簡介

本簡介供一般使用者參考，說明如何登入、收發私人郵件、看新聞、張貼公告與查用戶清單。

---

## 登入與登出

- **登入或註冊**：輸入 `/hi 帳號 密碼`
  - 帳號與密碼皆為 **4～8 個可顯示 ASCII** 字元（不含中文）。
  - 若帳號不存在則自動註冊；存在則驗證密碼登入。
  - 成功後會顯示歡迎訊息；若有新信會多一個 ✉️ 提示。

- **登出**：輸入 `/bye`
  - 會解除目前裝置與帳號的綁定，之後需再以 `/hi` 登入。

- **改密碼**：登入後輸入 `/pass 新密碼`
  - 新密碼同樣 4～8 字元、可顯示 ASCII。

---

## 私人郵件（/m 或 /mail）

**須先登入。**

- **看總數與最新幾則摘要**
  - 輸入 `/m` 或 `/mail`（不加數字）。
  - 會回覆總信數，以及**最新最多 5 則**的摘要（編號、寄件者、時間、內容預覽）。

- **讀第 N 則全文**
  - 輸入 `/m N` 或 `/mail N`（N 為正整數，依「摘要列表的編號」）。
  - 例如 `/m 1` 為最新一則全文，`/m 2` 為第二新的一則全文。

- **從第 N 則開始列摘要**
  - 輸入 `/m N-` 或 `/mail N-`（N 後面加一個減號 `-`）。
  - 表示「從第 N 則開始，往後列出摘要」，方便一次看較多筆而不只 5 筆。
  - 例如信很多時，先 `/m` 看最新 5 筆，再輸入 `/m 6-` 可看第 6 筆之後的摘要。

- **發私信給別人**
  - 輸入 `@用戶名 訊息內容`（以 @ 開頭，後接用戶名與內容，中間空格分隔）。
  - 用戶名須存在，訊息長度有限制，超過會提示無法送出。

## 新聞（/n 或 /news）與張貼（/p 或 /post）

**須先登入。**

- **看新聞總數與摘要**
  - 輸入 `/n` 或 `/news`：顯示總則數與**最新最多 5 則**摘要（編號、發文者、時間、內容預覽）。

- **讀第 N 則新聞全文**
  - 輸入 `/n N` 或 `/news N`（N 為正整數）。
  - 例如 `/n 1` 為最新一則全文。

- **從第 N 則開始列摘要**
  - 輸入 `/n N-` 或 `/news N-`（N 後加減號 `-`）。
  - 表示從第 N 則開始往後列摘要，用法與郵件 `/m N-` 相同。

- **張貼一則新聞**
  - 輸入 `/p 內文` 或 `/post 內文`。
  - 內文有長度上限，超過會提示無法張貼。

---

## 用戶列表（/users）

**須先登入。**

- 輸入 `/users`：列出所有用戶（用戶名、最後登入時間）。
- 輸入 `/users 關鍵字`：只顯示用戶名**包含**該關鍵字的用戶（不區分大小寫）。

---

## 其他

- **說明**：輸入 `/h`、`/help` 或 `/?`，可得到精簡指令列表。

---

## 進階用法整理

| 想做什麼           | 指令範例        | 說明 |
|--------------------|-----------------|------|
| 看郵件總數+最新 5 筆摘要 | `/m` 或 `/mail` | 無參數 |
| 讀「第 N 則」郵件全文   | `/m N`          | N = 1 為最新一則 |
| 從第 N 則開始列郵件摘要 | `/m N-`         | N 後面要加 `-` |
| 看新聞總數+最新 5 筆摘要 | `/n` 或 `/news` | 無參數 |
| 讀「第 N 則」新聞全文   | `/n N`          | N = 1 為最新一則 |
| 從第 N 則開始列新聞摘要 | `/n N-`         | N 後面要加 `-` |

以上為一般使用者會用到的功能；時間顯示皆為節點本地時間，單則訊息與列表行數有長度限制，超過時系統會提示。
